PROYECTO DESARROLLO REST API



Usuario

Resource Description
GET usuario/list Retorna una lista con todos los usario que es encuentren en esl sistema en formato xml.
GET usuario/datos/:id Retorna el nick de un usuario especifico se le pasa como parametro el id del usuario en formato XML.
GET usuario/obtenerFoto/:nick Retorna la ruta de la foto del usuario que le pasas por parametro.
POST usuario/insertar Se encarga de insertar el usuario en persistencia pasandole un xml y retornando el xml del usuario ya insertaod en persistencia con su id ya generad.
POST usuario/validar Se encarga de validar que el nick y la clave del usuario sean validas pasandole un xml con el nick y la clave y devolviendo todo los datos del usuario con su token ya asignado.
POST usuario/unico/:nick Retorna toda la informacion del usuario pedio, al metodo se le pasa como parametro el nick del usuario para ver toda su informacion.
POST usuario/modificar/:token Retorna el usuario con las nuevas modificaciones, al metodo se le pasa un xml con toda la informacion del usuarioy el token para complir con su peticion, para actualizarlo y se devuelve un xml del usuario con las modificaciones.
POST usuario/foto/:token Se encarga de registrar su foto de perfil en el sistema, pasandole por parametro el token y el XML del id del usuario y el nombre de la foto.
DELETE usuario/eliminar/:nick Se encarga de eliminar el usuario del sistema y devuelve un mensaje si elimino el usuario o si no pudo eliminarlo se le pasa como paramtro el nick del usuario a eliminar.


Comentario

Resource Description
GET comentario/listart Retorna una lista con todos los comentarios que es encuentren en esl sistema en formato xml.
GET comentario/listar/:id Retorna una lista con todos los comentarios hijos de un comentario en especificoen formato XML, se le pasa como parametro el id del comentario para buscar sus hijos
GET comentario/listaru/:nick Retorna todos los comentarios realizados por un usuario en especifico que se le pasa por parametro en formato XML
GET comentario/listar/tag/:tag Retorna todos los comentarios que esten asociados al tag que se le pasa por parametro en formato XML.
GET comentario/tags/:id_c Retorna una lista con todos los tags asociados al comentatio en formato XML se le pasa como parametro el id del comentario.
GET tags/listar Retorna una lista con todos los tags que se encuentren en sistema en formato XML.
GET comentario/obtenerAdjunto/:id_c Se encarga de obtener el adjunto el comentario que se le pasa como parametro.
POST comentario/insertar/:token Retorna el comentario ya insertado en persistencia con su id, en formato XML, se pasa como parametro el token del usuario que esta haciendo el comentario y un XML con el texto del comentario, la privacidad del comentario, el id del usuario creador el comentario, el padre, es decir, si esta respondiendo a un comentario ya realizado antes o un nuevo comentario y se le puede pasar de una vez los tags al comentario si es padre si no, no se le pasa tags.
POST comentario/like/:token Retorna el comentario en XML al cual se le ha hecho la peticion de like o dislike, se le pasa como parametro el token del usuario el cual realixa la peticion, en el XML se le pasa el id del comentario, el id del usuario y la peticon de like o dislike.
POST comentario/insertarAdjunto/:token Se encarga de insertar el adjunto del comentario en persistencia pasandole como parametro el token del usuario y por XML el mismo XML de el insertar comentario pero agregandole la etiqueta de la ruta.
DELETE comentario/eliminar/:nick/:id_c/:token Retorna un mensaje diciendo que se elimino el mensaje, ha este metodo se le pasa como parametro el nick del usuario, el id del comentario, y el token del usuario. este metodo elimina los comentarios que sean del usuario o si el usuario es dueno del comentario padre puede eliminar cualquier hijo a ese comentario.
