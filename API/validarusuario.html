<API

BLOG

POST usuario/validar

Direccion URL

http://localhost:9000/blog/usuario/validar

Descripcion

Se encarga calidar el usuario y asignarle un token, se le pasa un XML. si no se pasa un XML el framework lanza un erro HTTP 400 ya que la funcion por defecto siempre esperara un XML

 

Entradas XML

<usuario>
<nick>lucia2</nick>
<clave>1234</clave>
</usuario>

Salidas XML

<usuario>
<id__u>8</id__u>
<objectid>4fc81679fe716c1eeaaa28bb</objectid>
<nombre>Lucia</nombre>
<nombres>Ana</nombres>
<apellido>Cardosso</apellido>
<apellidos>Militao</apellidos>
<email>lucia@hotmail.com</email>
<nick>lucia2</nick>
<pais>Venezuela</pais>
<biografia>Desarrollando</biografia>
<sexo>M</sexo>
<fecha>21/06/1987</fecha>
<clave>1234</clave>
<token>414490756</token>
</usuario>
